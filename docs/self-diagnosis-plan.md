# ASC 자기 진단 검사 — 설계 초안 (v1)

> 목적: ASC 멤버가 **"지금 나의 상황과 상태"** 를 진단받는 검사. 결과 = 상태 리포트(主) + 어울리는 트랙 추천(곁다리).
> 성격: 온보딩(빠른 트랙 매칭)이 아니라 **자기 이해 도구.** 길이는 짧을 필요 없음(약 35~40문항).
> ⚠️ 전제 주의: **모두가 창업가라고 단정하지 않는다.** 직장인·초기 사업가·전문직(기존 사업 활력)·탐색자가 섞여 있으므로, 검사 맨 앞에서 **상황 유형**을 먼저 묻고 나머지 진단을 그 맥락에 비춰 해석한다.

## 구조 요약
**① 상황 유형(맥락 렌즈, 4택) + ②~⑥ 진단 축 5개(누구에게나 공통).**
①은 점수화하지 않고 결과 해석·트랙 추천의 렌즈로 쓴다.
> 방법: 각 축을 학계 검증 척도에서 빌려와 ASC 맥락으로 각색. (McGee ESE 2009 / Grit-S / SDT MWMS / Hollenbeck 목표몰입 / EntreComp / 자원슬랙)

## 응답 방식
- 대부분 **5점 리커트** (1 전혀 아니다 … 5 매우 그렇다). ③은 자신감 정도(1~5).
- **① 단계만 단일 선택**(척도 아님).

## 6개 축 × 문항 초안

### ① 상황 유형 (단일 선택, 맥락 렌즈 — 점수화 X)
> 창업 사다리(단계) 대신 "지금 내가 어떤 맥락에 서 있나"를 묻는다. 결과 해석·트랙 추천을 이 유형에 맞춰 다르게.

지금 나의 상황에 가장 가까운 것은?
- **A. 직장인 준비형** — 회사 다니며 독립·부업·이직을 준비 중 (원하는 것: 뭘/어떻게 시작할지)
- **B. 초기 사업가** — 이미 내 사업·프리랜스를 굴리는 중 (원하는 것: 더 키우기, 매출)
- **C. 전문직·기존사업 활력형** — 의사·세무사·강사 등 이미 자기 일이 있음 (원하는 것: 세일즈·퍼스널 브랜딩으로 활력)
- **D. 탐색형** — 아직 뭘 할지 모름 (원하는 것: 방향부터 잡기)

### ② 동기 & 방향 명확성 (8문항)
> 근거: SDT 다차원동기척도(자율 vs 통제 동기) + Hollenbeck 목표몰입.

**동기 유형 (왜 하는가)**
- (자율) 이 일 자체가 재미있고 의미 있어서 한다
- (자율) 내가 중요하게 여기는 가치와 맞아서 한다
- (통제) 안 하면 뒤처질 것 같아 불안해서 한다
- (통제) 남들이 하니까 / 인정받고 싶어서 한다

**방향 명확성**
- 내가 뭘 이루고 싶은지 분명하다
- 지금 목표를 위해 뭘 해야 하는지 안다
- 이 목표에 진심으로 몰입하고 있다
- 목표가 자주 흔들리거나 막막하다 (역문항)

### ③ 자기효능감 (10문항)
> 근거: McGee 2009 ESE 19문항 → 5국면 × 2로 추림. "할 자신이 있다" 정도.

- (검색) 새로운 제품·서비스 아이디어를 떠올릴 자신이 있다
- (검색) 시장의 기회를 알아볼 자신이 있다
- (계획) 고객이 원하는 걸 파악할 자신이 있다
- (계획) 목표를 세우고 계획을 짤 자신이 있다
- (동원) 필요한 사람·자원을 끌어모을 자신이 있다
- (동원) 내 아이디어를 남에게 설득할 자신이 있다
- (실행-사람) 함께 일할 사람을 이끌 자신이 있다
- (실행-사람) 관계를 맺고 네트워크를 활용할 자신이 있다
- (실행-재무) 돈 관리를 할 자신이 있다
- (실행-재무) 수익 낼 방법을 만들 자신이 있다

### ④ 끈기 (4문항 — 노력 지속만)
> 근거: Grit-S 중 '노력 지속(perseverance of effort)' 하위요인만. '흥미 일관성'은 ASC 탐색 철학과 충돌해 제외.

- 한번 시작한 일은 끝까지 해낸다
- 어려움이 있어도 포기하지 않고 계속한다
- 몇 달, 몇 년 걸리는 목표도 꾸준히 밀어붙인다
- 좌절해도 금방 다시 일어나 계속한다

### ⑤ 역량 자산 — 내 무기 (7문항, ASC 커스텀)
> 근거: EntreComp + Man/Chandler 역량차원을 ASC 7트랙 스킬 도메인으로 재구성. **트랙 추천 엔진.**

- (콘텐츠) 글·영상 등 콘텐츠로 사람을 모으는 데 강하다
- (빌드) 노코드/코드로 제품·서비스를 만드는 데 강하다
- (앱) 앱을 기획하고 만드는 데 관심·역량이 있다
- (세일즈) 물건·서비스를 파는(마케팅·영업) 데 강하다
- (자동화) AI·자동화로 일을 굴러가게 만드는 데 강하다
- (기술심화) 개발·터미널·깃허브 같은 게 익숙하다
- (방향탐색) 아직 내 무기가 뭔지 잘 모르겠다 (→ '나 탐구' 신호)

### ⑥ 리소스 & 제약 (5문항)
> 근거: 지각된 자원 슬랙(시간·자본·인적·사회자본).

- 지금 이 일에 쓸 수 있는 시간이 충분하다
- 당분간 버틸 수 있는 여유 자금이 있다
- 도움을 청할 수 있는 사람·인맥이 있다
- 이미 팔 수 있는 것(제품·지식·청중)이 있다
- 실패해도 재정적으로 크게 타격받지 않는다

**소계: 1 + 8 + 10 + 4 + 7 + 5 = 35문항** (필요 시 축당 1~2개 보강 → 40)

## 채점 & 결과 리포트 (스케치)

### 채점
- 각 축 = 해당 문항 평균 → 0~100 환산 (역문항 뒤집기). ①은 단계 값.
- ⑤ 역량은 도메인별 개별 점수 유지(트랙 매핑용).

### 결과 화면 구성
1. **한 줄 요약**: "『(상황 유형)』인 당신, 지금 강점은 『(최고 축)』 / 키울 곳은 『(최저 축)』" — **단계로 단정하지 않고 상황+강약점으로 서술**
2. **5축 프로파일**: 레이더 차트 or 막대 — 강점/성장 포인트 한눈에
3. **축별 짧은 해석 (상황 맞춤)**: 같은 점수라도 상황 유형별로 다르게 해석
   - 예) ⑥ 자원(시간) 낮음 → A(직장인)면 "정상, 짧은 사이클로" / B(사업가)면 "시간 확보가 급선무"
4. **곁다리 — 어울리는 트랙 1~2개** + 이유 + [신청하러 가기] 버튼

### 트랙 추천 규칙 (곁다리, 초안)
**상황 유형(①)으로 큰 방향을 잡고, ⑤ 우세 스킬 + ② 동기로 세부 트랙 결정.**
- 상황별 성향(prior):
  - **C. 전문직·활력형** → 세일즈 / 크리에이터(퍼스널 브랜딩) 우선
  - **D. 탐색형** → 나 탐구 우선
  - **A. 직장인 준비형 / B. 초기 사업가** → ⑤ 무기에 따라 결정
- ⑤ 무기 기반 세부:
  - 콘텐츠 최상 → 크리에이터
  - 빌드 최상 + 기술심화 낮음 → 빌더 기초
  - 빌드/기술심화 최상 → 빌더 심화
  - 앱 최상 → 앱 개발
  - 세일즈 최상 + 팔 것 있음(⑥) → 세일즈
  - 자동화 최상 → AI 에이전트
  - 방향탐색 높음 / 방향 명확성(②) 낮음 → 나 탐구
