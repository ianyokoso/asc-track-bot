ASC 트랙 봇 — 운영자 테스트 체크리스트

실전 오픈 직전, 운영자가 직접 눈으로 확인하는 기능 점검표

대상: 운영자 (비개발자) · 작성일: 2026-05-18

🎯 가장 중요한 3가지 — 이것만은 꼭

  1. 봇이 메인 ASC 디스코드 서버에 채널을 만들지 않는지 — 시범 1회 돌린 뒤 메인 서버 사이드바에 새 카테고리/채널이 안 생겼는지 눈으로 확인
  2. 노션 멤버 DB에 같은 사람이 두 번 안 생기는지 — 신청자 본인 row 1개만 있고 "트랙" 컬럼만 채워졌는지 확인
  3. 모바일에서 디스코드 로그인 → 신청 제출이 끝까지 되는지 — 학생 절반은 모바일

1. 신청 페이지가 학생에게 보일 때 눈으로 체크

직접 신청 URL을 열어보면서 학생 시점으로 확인

  • 신청 페이지 URL 접속 → 카운트다운 또는 신청 폼이 정상 표시
  • 디스코드 로그인 버튼 클릭 → 로그인 후 폼 진입
  • 모바일 핸드폰으로도 같은 흐름 한 번 (학생 절반은 모바일로 신청)
  • 빌더 기초/실전 트랙 = "리뉴얼 중" 카드로 보이고, 선택 불가
  • 요일 슬롯 확인: 화요일 = AI 에이전트, 수요일 = 나 탐구
  • OT 날짜 = 5/24 로 표기
  • 크리에이터 트랙 선택 시 숏폼/롱폼 선택 박스 뜸

2. 신청 제약이 제대로 걸리는지

  • 정규 트랙 3개째 선택하면 막힘 (최대 2개 안내 메시지)
  • 라이트 트랙 = 크리에이터/빌더 각각 1개씩만 선택됨
  • 크리에이터 선택 후 숏폼/롱폼 안 고르고 제출 → 제출 차단 + 안내
  • 조장 옵션이 비활성 상태일 때 클릭해도 안 눌림

3. 신청 제출 → 데이터 정상 저장

  • 본인 디스코드로 신청 1건 제출 → "신청 완료" 화면 뜸
  • 같은 계정으로 다시 들어가면 본인이 낸 신청 내역 그대로 보임
  • 신청 수정 가능 (트랙 바꿔서 재제출)
  • 신청 취소/드롭 가능
  • 노션 트랙 신청 DB 열어보면 → 방금 낸 신청이 row로 들어와 있음

4. 운영자 대시보드 — 클릭해보면서 확인

  • 운영자 페이지 진입 → 신청자 목록 정상 표시 (이름, 디스코드, 트랙)
  • 트랙별로 신청자 인원수가 신청 DB와 일치
  • 한 명을 다른 조로 드래그 → 화면에 반영
  • "조 배정 완료" 버튼 클릭 → 로딩 → 완료 메시지

5. "조 배정 완료" 누른 후 — 봇이 자동으로 한 일 핵심

운영 트랙 디스코드 서버 열어서 눈으로 확인

  • 트랙별 카테고리 생김 (예: =====크리에이터 트랙=====)
  • 카테고리 안에 공지 채널, 과제-인증 채널 생김
  • 조별 채널 생김 (예: 크리에이터-9기-1조)
  • 화상미팅 음성 채널 생김 (예: 크리에이터-9기-1조-화상미팅)
  • 조장 채널 생김

6. 학생 시점에서 확인 — 본인 디스코드로 들어가보기

  • 본인에게 트랙 역할이 자동 부여됨 (예: 크리에이터-9기-1조)
  • 본인 조 채널이 사이드바에 보임
  • 다른 조 채널은 안 보임 (격리 확인)
  • 화상미팅 채널 들어가서 마이크/카메라/화면공유 가능
  • 라이트 트랙 신청자는 부모 트랙 공지/과제만 보이고, 조 채널은 안 보임

7. 노션 측 확인 — "조 배정 완료" 누른 후

  • 트랙·조 마스터 페이지 안에 트랙별 inline DB 자동 생성됨
  • 그 안에 조별 멤버 목록 들어가 있음
  • 운영봇 멤버 마스터 DB → 본인 row의 "트랙" 컬럼이 채워져 있음
  • 멤버 DB에 본인 row가 새로 또 만들어지지 않았는지 (중복 X) — 가장 중요
  • 매칭 실패한 멤버가 있으면 운영자 페이지에 "마스터 DB 추가" 버튼 떠 있음

8. 사고 대응용 명령어 — 한 번씩 눌러보기

가짜 기수(예: 99기)로 미리 테스트하고 진짜 운영 가기 전 정리

  • 디스코드에서 !채널삭제 99 → confirmation 메시지 뜸 → 확인 → 99기 채널·역할만 삭제, 다른 기수는 그대로
  • !역할삭제 99 → 채널은 살아있고 역할만 삭제됨
  • 운영자가 아닌 사람이 같은 명령어 치면 → 거부됨

9. 마감 후 화면 — 시간 지나면 자동 전환되는지

  • 신청 마감 시각 지나면 페이지가 "마감되었습니다" 화면으로 자동 전환
  • 이미 신청한 사람은 본인 신청 내역 보임
  • 미로그인 사용자는 로그인 버튼만 보임 (폼 X)

🟢 추천 시범 운행 시나리오 (30분이면 됨)

  1. 본인 + 운영진 1명, 본인 핸드폰으로 신청 페이지 접속 → 로그인 → 신청 제출
  2. 운영자 페이지에서 두 사람을 다른 조로 배정 → "조 배정 완료"
  3. 운영 트랙 디스코드 서버 가서 → 카테고리·채널·역할·화상미팅 생긴 거 눈으로 체크 (§ 5)
  4. 학생 시점에서 → 본인 조만 보이는지 체크 (§ 6)
  5. 노션 가서 → 신청 row · inline DB · 멤버 DB 트랙 컬럼만 갱신됐는지 체크 (§ 7)
  6. 메인 ASC 디스코드 서버 사이드바 확인 → 새로 생긴 거 0개 (가장 중요 1번)
  7. 시범 자원 정리 → !채널삭제 99 또는 디스코드에서 우클릭 삭제

→ 이 7단계 다 통과하면 내일 실전 OK입니다.
